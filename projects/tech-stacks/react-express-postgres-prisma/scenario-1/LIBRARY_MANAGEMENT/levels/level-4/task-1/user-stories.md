# User Stories

## Story 1: Review Full Borrowing History
As a librarian,
I want Borrow Records to show all borrowing activity,
So that I can audit active and returned items from one screen.

## Story 2: Focus on Active Loans Quickly
As a librarian,
I want to filter Borrow Records by borrowed and returned status,
So that I can quickly separate open loans from completed returns.

**Acceptance Summary:**
- [ ] AC-1: Backend returns complete borrow-history rows needed by the Borrow Records UI
- [ ] AC-2: Backend supports status filtering for BORROWED, RETURNED, and ALL
- [ ] AC-3: Borrow Records UI supports the same status filters and keeps an empty-state output
- [ ] AC-4: Borrow Records UI renders status/action output consistently for BORROWED and RETURNED
